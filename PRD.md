## 1. 执行摘要

### 1.1 产品愿景

**我敲 (TapCare)** 是一款运行在 HUAWEI GT 系列智能手表上的健康练习应用，通过高精度传感器算法帮助用户进行中医经络敲打练习，提供准确的计数、力度识别和练习记录功能。

### 1.2 核心价值主张

- **精准检测**: 多特征融合算法（加速度 + 陀螺仪），准确率 > 95%
- **智能识别**: 自动区分敲打动作与日常手势（甩手、拍手等）
- **个性化校准**: 左右手自动识别，自适应个人敲打习惯
- **数据驱动**: 完整的练习记录和统计分析，支持长期健康追踪

### 1.3 成功指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 敲打检测准确率 | ≥ 95% | 真机测试对比人工计数 |
| 误识别率 | ≤ 3% | 日常动作干扰测试 |
| 用户留存率（7 天） | ≥ 60% | 应用使用统计 |
| 平均练习时长 | ≥ 5 分钟/次 | 会话数据统计 |
| 应用启动时间 | < 2 秒 | 性能监控 |

## 4. 功能需求

### 4.1 敲打检测功能

#### 4.1.1 功能描述

通过手表内置的加速度计和陀螺仪传感器，实时检测用户的敲打动作，并准确计数。

#### 4.1.2 功能需求

| 需求 ID | 需求描述 | 验收标准 | 优先级 |
|---------|----------|----------|--------|
| FD-001 | 实时检测敲打动作 | 检测延迟 < 100ms | P0 |
| FD-002 | 区分敲打与其他手势 | 误识别率 < 3% | P0 |
| FD-003 | 识别力度等级（轻/中/重） | 力度识别准确率 > 90% | P1 |
| FD-004 | 支持左右手佩戴识别 | 自动校准准确率 > 95% | P1 |
| FD-005 | 防误触机制 | 日常活动不误触发 | P0 |

#### 4.1.3 技术规格

**检测算法参数**:

| 参数 | 值 | 说明 |
|------|-----|------|
| 加速度阈值 | 15.0 m/s² | 触发检测的最小加速度变化 |
| 陀螺仪阈值 | 2.0 rad/s | 判定手腕稳定的阈值 |
| 防抖时间 | 200ms | 两次敲打的最小间隔 |
| 敲打时长范围 | 50-300ms | 有效敲打的持续时间范围 |
| 峰值加速度 | > 18 m/s² | 有效敲打的最小峰值 |
| 传感器采样率 | 20Hz | 50ms 间隔采样 |

**状态机设计**:

```
┌─────────┐    加速度突变     ┌──────────────┐
│  空闲   │ ───────────────▶ │  敲打进行中   │
│ (Idle)  │   且陀螺仪稳定    │  (Tapping)   │
└─────────┘                  └──────────────┘
     ▲                              │
     │                              │ 加速度恢复正常
     │    时长有效 + 峰值达标        │ 或超时
     └──────────────────────────────┘
               │
               ▼
      ┌──────────────┐
      │   输出敲打    │
      │   (TapEvent) │
      └──────────────┘
```

### 4.2 会话管理功能

#### 4.2.1 功能描述

#### 4.2.2 功能需求

| 需求 ID | 需求描述 | 验收标准 | 优先级 |
|---------|----------|----------|--------|

| SM-002 | 暂停会话 | 暂停期间不计数 | P0 |
| SM-003 | 恢复会话 | 从暂停处继续计数 | P0 |
| SM-004 | 完成会话 | 达到目标次数自动完成 | P0 |

**数据库表结构**:

```sql

CREATE TABLE tap_records (
  id TEXT PRIMARY KEY,
  timestamp INTEGER NOT NULL,
  part TEXT NOT NULL,
  force INTEGER NOT NULL,
  force_level TEXT NOT NULL,
  confidence REAL NOT NULL,
  session_id TEXT NOT NULL,
  accel_x REAL,
  accel_y REAL,
  accel_z REAL
);

-- 会话表
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  start_time INTEGER NOT NULL,
  end_time INTEGER,
  total_duration INTEGER NOT NULL,
  pause_duration INTEGER NOT NULL,
  part TEXT NOT NULL,
  target_count INTEGER NOT NULL,
  actual_count INTEGER NOT NULL,
  completed INTEGER NOT NULL
);
```

### 5.2 模块设计

#### 5.2.1 TapDetector（敲打检测器）

**职责**: 传感器数据处理和敲打动作识别

**核心方法**:
```typescript
processSensorData(accel, gyro): TapEvent | null
calibrate(accel): void
setWearingHand(hand): void
reset(): void
```

**输入**: 加速度计数据、陀螺仪数据  
**输出**: TapEvent（包含时间戳、力度、置信度）

#### 5.2.2 SessionManager（会话管理器）

**职责**: 管理练习会话的状态转换和数据追踪

**核心方法**:
```typescript
transition(event: SessionEvent, payload?: any): void
setTarget(count: number): void
setOnStateChange(callback): void
getData(): SessionData
```

**输入**: 用户操作事件、敲打事件  
**输出**: 会话状态、练习数据

#### 5.2.3 CalibrationService（校准服务）

**职责**: 执行校准流程，采集基准数据

**核心方法**:
```typescript
startCalibration(): Promise<CalibrationResult>
detectWearingHand(): Promise<'left' | 'right'>
```

**输入**: 传感器原始数据  
**输出**: CalibrationResult（包含佩戴手、基准加速度）

#### 5.2.4 DataStorageService（数据存储服务）

**职责**: 数据持久化和查询

**核心方法**:
```typescript
initialize(context): Promise<void>
saveTapRecord(record): Promise<void>
saveSession(session): Promise<void>
getDailySummary(date): Promise<DailySummary | null>
```

**输入**: 数据对象  
**输出**: 存储结果、查询结果

## 6. 用户界面

### 6.1 设计原则

1. **简洁优先**: 小屏设备，信息密度适中
2. **触控友好**: 按钮尺寸 ≥ 44x44 像素

## 11. 风险评估

### 11.1 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 传感器检测准确率不达标 | 中 | 高 | 多轮算法调优，增加校准流程 |
| 不同手表型号兼容性问题 | 中 | 中 | 多设备测试，适配不同传感器 |
| 后台运行电量消耗过大 | 高 | 中 | 优化采样率，使用低功耗模式 |
| 数据存储空间不足 | 低 | 低 | 自动清理旧数据，限制存储量 |

### 11.2 产品风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 用户留存率低 | 中 | 高 | 增加提醒功能， gamification 元素 |
| 误识别导致用户体验差 | 中 | 高 | 持续优化算法，提供灵敏度调节 |
| 功能过于简单，缺乏吸引力 | 低 | 中 | 快速迭代，增加数据统计和社交功能 |
| 竞品抢先发布 | 低 | 低 | 专注产品质量，差异化竞争 |

| 风险 | 概率 | 影响 | 缓解措施 |

### 11.4 应急预案

- 招募更多测试用户提供数据

- 紧急修复版本
